Excursium 

Автоматизированное тестирование — тесты для критических бизнес-процессов (регистрация, авторизация, бронирование, фильтрация).

Инструменты для тестирования: Selenium для автоматизированного тестирования пользовательского интерфейса.

Авторизация: Тест берет email и password из файла .env и производит авторизацию. Тест пройден если авторизация успешна или если сайт выдал ошибку "Неверная почта или пароль". Можно проводить как позитивные так и негативные тесты, изменяя логин и пароль в файле .env

Бронирование: Тест берет email, password, страницу с экскурсией, дату, телефон, имя и примечания из файла .env и производит авторизацию, затем переходит на страницу с экскурсией, заданную в файле и производит бронирование по данным из файла. Выбраны все каналы связи для бронирования. Тест пройден если авторизация успешна, бронирование произведено, сообщение "Спасибо! Заявка принята" появилось. В случае негативного тестирования тест провалится из за таймаута.и появления сообщения: ВВедите коректный ...........

Регистрация: Тест берет данные из файла .env и производит регистрацию и заполняет все формы знакомства. При позитивном сценарии (email новый, есть доступ к коду) тест будет успешен, при негативном сценарии тест провалится. В отчете будет указано на какой страницы регистрации. 

Отчет о тестировании предоставлен в HTML документе report.

В фаиле .evn содержится набор как позитивных так негативных сценариев для автотеста.
