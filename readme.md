Автоматизированное тестирование — тесты для критических бизнес-процессов (регистрация, авторизация, бронирование, фильтрация).

Авторизация: Тест берет email и password из файла .env и производит авторизацию. Тест пройден если авторизация успешна или если сайт выдал ошибку "Неверная почта или пароль"

Бронирование: Тест берет email, password, страницу с экскурсией, дату, телефон из файла .env и производит авторизацию, затем переходит на страницу с экскурсией, заданную в файле и производит бронирование по данным телефона и даты из файла. В качестве имени использовано фиксированное - Наташа, в примечании написанно: Тест, выбраны все каналы связи для бронирования. Тест пройден если авторизация успешна, бронирование произведено, сообщение "Спасибо! Заявка принята" появилось.

Регистрация: Тест берет email и password из файла .env и производит регистрацию до момента ввода проверочного кода из email. Если загружается страница ввода кода - тест пройден. Так же проверяется уже зарегистрированный email. Если email зарегистрирован, то тест тоже пройден. 
Примечания:
*Хрупкие селекторы. Использую длинные цепочки т.к. ID, class и поиск по тексту нет работают.
